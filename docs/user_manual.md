# Lighthouse 사용자 설명서 (User Manual)

사내 IT 지원 시스템 "Lighthouse"의 실제 UI(웹 애플리케이션)를 기반으로 사용자(일반 사원)와 관리자(IT 팀)의 사용 방법을 설명합니다.

---

## 🔑 접근 및 로그인

현재 시스템은 MVP 단계를 위해 빠른 테스트 계정 연동(NextAuth Credentials)을 지원합니다.  
로그인 페이지(`/login`)에 접속하여 권한별 테스트 계정을 이용해 로그인하세요.

*   **관리자 (IT 팀)**: `test-admin@example.com` / `mockpassword`
*   **일반 사원 (User)**: `test-user@example.com` / `mockpassword`
*   *(실제 운영 환경에서는 사내 SSO 계정 연동으로 대체됩니다.)*

---

## 👨‍💼 일반 사원 (User) 기능 가이드

### 1. 대시보드 한눈에 보기
로그인 후 처음 보이는 **대시보드(`/`)** 상단에서는 내가 접수한 티켓들의 현황을 카드 형태로 빠르게 파악할 수 있습니다. 
하단 **[최근 티켓 현황]** 데이터 테이블에서 각 티켓을 클릭하면 상세 처리 페이지로 넘어갑니다.

### 2. 새 IT 지원 요청 (티켓 생성)
1.  대시보드 우측 상단의 **[새 티켓 작성 (Plus 아이콘)]** 버튼을 클릭합니다.
2.  문의하려는 내용에 가장 잘 맞는 **분류(Category)** 와 **우선순위** 를 선택합니다.
3.  도움이 필요한 내용을 텍스트 폼에 입력합니다. 필요한 경우 에러 캡처 화면 파일 등을 드래그 앤 드롭으로 등록할 수 있습니다.
4.  [등록하기]를 누르면 담당 부서 접수 대기 상태(`OPEN`)로 티켓 목록에 반영됩니다.

### 3. 진행상황 확인 및 문제 해결
*   내 티켓을 클릭하여 **[상세 조회 화면 (`/tickets/[id]`)]** 으로 들어오면 현재 티켓의 `Status(상태)` 정보와 `Priority(우선순위)` 뱃지를 볼 수 있습니다.
*   우측 패널에서 현재 이 업무를 살펴보고 있는 **담당 관리자** 가 누구인지, 접수된 시간은 언제인지 투명하게 확인 가능합니다.
*   가운데 위치한 **[대화 기록 (Comments)]** 영역을 마치 메신저처럼 활용하세요. IT 부서에서 남긴 작업 현황을 보거나, "파일을 다시 첨부해 드립니다" 와 같은 내용과 파일을 추가할 수 있습니다.

## 🛠️ IT 관리자 (Admin) 전용 가이드

### 1. 전사 티켓 모니터링 (대시보드)
*   일반 사용자와 달리 관리자의 대시보드 카드에는 가장 중요한 **[미배정 티켓 (붉은색 카드)]** 섹션이 추가로 노출됩니다.
*   관리자 뷰 하단의 데이터 테이블은 내가 쓴 글 뿐만 아니라 "시스템 전체의 티켓 목록"을 보여줍니다. 
*   제공되는 Analytics 차트를 통해 금주 부서별 IT 리소스 소모량 통계를 볼 수 있습니다.

### 2. 티켓 담당자 배정 & 상태 컨트롤
*   [미배정]인 티켓을 클릭하여 진입하면, 우측 하단에 파란색의 **[관리자 제어 패널]** 이 노출됩니다.
*   본인 혹은 다른 IT 부서 동료를 `담당자` 로 즉시 지정(배정)할 수 있습니다.
*   업무 작업을 시작하면서 `상태(Status)` 를 `진행중(IN_PROGRESS)` 으로 변경해주어 사용자가 안심할 수 있게 합니다.

### 3. 처리 이력 관리 및 히스토리
*   관리자 제어 패널에서 내가 버튼을 눌러 상태나 담당자를 바꿀 때마다 시스템 뒷단(`TicketHistory`)에는 누가 언제 무엇을 바꿨는지 투명하게 문책 및 통계 용도로 영구 저장됩니다.
*   작업이 모두 마무리되면 상태를 `해결됨(RESOLVED)` 혹은 `종료(CLOSED)` 로 변경하여 본인의 처리 티켓 실적 메트릭스를 높이세요.
